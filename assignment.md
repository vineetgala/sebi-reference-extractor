Knowledge Graph Agent 
You are working with the Head of Compliance for a major Indian bank. They are tasked with making sure that the bank is compliant with directions listed in SEBI circulars (https://www.sebi.gov.in/sebiweb/home/HomeAction.do?doListing=yes&sid=1&ssid=7&smid=0), and also able to respond to SEBI, customers, and intermediaries regarding the bank's compliance posture. One major challenge is that most circulars reference other documentation, i.e. circulars, regulations, laws etc. and since they are all in PDF format it is difficult to easily reference clauses in older documents. You are tasked with building an AI agent to simplify this effort to track and comply with SEBI’s evolving guidelines and directives. 



Deliverables 
Create an agent that takes as its input one PDF file and is able to share all references made to other documentation. Please share this agent and instructions on how to run it (in form of a public github repo). Make sure to capture titles of the circulars as well as page numbers. Feel free to use an LLM provider to complete this challenge, Google Gemini API has a generous free tier

In your submission, include the use of evaluations to improve the agent’s outputs. Bonus points for showing improvements in your agent due to your evaluation methodology

Please record and share a 2 minute video which covers the following:

Which models, tools, and APIs you used to solve the problem

Your high level approach on the different steps required to get good results, including (but not limiting to) evaluations

Any limitations that need to be addressed in v2 of your agent

Ideas on how this could scale to all PDF circulars issued by SEBI in order to create a useful knowledge graph
