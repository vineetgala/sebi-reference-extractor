## What this project is

Hyde take-home assignment. Build an agent that takes one SEBI circular PDF and extracts all references to other documents (circulars, master circulars, regulations, acts, notifications), with titles and source page numbers.

The assignment also explicitly requires evaluations that demonstrate improvement. That work is done and the results are saved.

---

## Key commands

```bash
# Extract references from a single PDF (deterministic only)
python3 agent-work/extract_references.py path/to/circular.pdf

# Extract from a directory (deterministic only)
python3 agent-work/extract_references.py pdfs/

# Extract with Gemini AI discovery (finds references regex missed — cross-paragraph splits,
# non-hardcoded acts, non-standard phrasing)
python3 agent-work/extract_references.py pdfs/ --use-ai

# Launch the API server (Swagger UI at http://localhost:8000/docs)
pip install -r api/requirements.txt
uvicorn api.server:app --reload --port 8000

# Launch the reference viewer (open http://localhost:7890/viewer/)
python3 viewer/serve.py

# Convert extractor output → eval scorer format
python3 evals/make_predictions.py

# Score predictions against ground truth
python3 evals/evaluate.py --gold-dir evals/ground_truth --pred-dir evals/predictions

# Full pipeline in one go
python3 agent-work/extract_references.py pdfs/ --use-ai && python3 evals/make_predictions.py && python3 evals/evaluate.py --gold-dir evals/ground_truth --pred-dir evals/predictions
```

## Architecture

**Deterministic-first, AI-optional.**

### 1. PDF parsing (`structured_pdf_extract.py`)
- Uses pdfminer for per-line layout extraction, pypdf for page count
- Output: `{pages: [{page_number, paragraphs: [{paragraph_id, text, ...}]}]}`
- All vendor deps are in `agent-work/vendor/` — no pip install needed
- Known limitation: does NOT join text across paragraph boundaries. If a title and its year are on separate lines, the regex won't find it (SCCR Regulations, 2018 in `stock_broker` PDF is the only known case affected).

### 2. Regex extraction (`extract_references.py`)
Four compiled patterns handle the main reference types in SEBI circulars:
- `MASTER_CIRCULAR_RE` — master circulars with topic body and date
- `SEBI_CIRCULAR_RE` — identifier-coded circulars (`CIR/XXX/YYY/ZZZZ`)
- `FORMAL_INSTRUMENT_RE` — Acts and Regulations with year (handles both singular "Regulation" and plural "Regulations")
- `NOTIFICATION_RE` — gazette notifications with date only

### 3. Registry + alias tracking
- `Registry` class deduplicates documents across all pages of a PDF
- When a full title is first seen (e.g. "SEBI (Credit Rating Agencies) Regulations, 1999 (CRA Regulations)"), the parenthetical alias ("CRA Regulations") is registered
- `alias_references_for_paragraph` scans later paragraphs for the short alias and back-resolves to the full document record
- Generic "Master Circular" alias is intentionally kept — all current PDFs reference only one master circular each, so cross-contamination risk is zero. If a PDF ever references two different master circulars, this could cause alias confusion (known design debt).

### 4. Locator extraction
- Preceding text before each match is scanned for section/para/regulation/clause/annexure numbers
- Stored in `target_locators` per mention — tells you *where inside* the referenced doc the circular is pointing

### 5. AI discovery (`--use-ai`)
- Fires once per document after regex extraction completes
- Full document text is sent to Gemini, page-by-page (paragraphs joined with spaces so cross-paragraph title fragments are continuous)
- Already-found regex records are listed in the prompt — Gemini returns only NEW references
- `temperature: 0.1` for near-deterministic output
- Structured JSON response: `document_type`, `title`, `identifier`, `year_or_date`, `source_page`, `evidence_text`, `exact_quote`
- Post-processing filters: self-references dropped; locator strings in the `identifier` field cleared (e.g. "Regulation 51" → null, kept if title exists); truncated act titles dropped; AI-discovered notifications with no title dropped
- Discovered records get `title_source: "ai_discovered"` and are merged into the registry like any other record

## Eval design

The scorer (`evals/evaluate.py`) matches predictions to gold using:
1. Explicit `canonical_key` if provided
2. Normalized `official_identifier` (strips non-alphanumeric, uppercases)
3. Normalized `canonical_title` / `title` (unicode normalize, lowercase, replace "SEBI" expansions)

**Primary metrics used in the submission:**
- `doc_precision`, `doc_recall`, `doc_f1` — unique documents found
- `page_precision`, `page_recall`, `page_f1` — unique (document, source_page) pairs
- `title_exact_recall` — % of gold docs where predicted title normalizes to the gold canonical
- `title_presence_recall` — % of gold docs where any non-empty title was predicted
- `type_accuracy_on_matched_docs` — % of matched docs with correct document_type

**Ground truth conventions:**
- `scored_references` — what the system must find
- `abstain_mentions` — what the system must NOT emit (self-references, generic phrases, bare notifications)
- Source pages always refer to pages in the *input PDF*, not pages in the referenced document

## Fixes made via eval feedback (the "before/after" story)

These are the 5 specific fixes driven by running `evaluate.py` on the initial extractor:

| Fix | What the eval revealed | What changed |
|---|---|---|
| 1. Singular Regulation pattern | CRA fixture Doc F1 66.7% — entire SEBI (Credit Rating Agencies) Regulations, 1999 missing | `FORMAL_INSTRUMENT_RE` changed from `Regulations` → `Regulations?` |
| 2. `infer_document_type` singular | CRA document had `type: "other"` not `"regulations"` | Changed from `"regulations" in lowered` to `re.search(r"\bregulations?\b", lowered)` |
| 3. Master circular title acronym | `ease_of_doing_investment_loc` Doc F1 80% — RTA circular matched as EXTRA with `("RTAs")` in title | Strip trailing parenthetical acronyms from body in `title_from_master_circular` |
| 4. Notification precision filter | `guidelines_for_custodians` had 2 FP notifications — Doc F1 85.7% | `make_predictions.py` skips `document_type: notification, title_source: generic_only` |
| 5. Date in circular short_title | Title exact recall low for identifier-only circulars | `short_title` changed from `"SEBI Circular {id}"` to `"SEBI Circular {id} dated {date}"` |

## Known gaps (not fixed, documented)

1. **Title exact recall 66.7% for `guidelines_for_custodians`** — two unresolved mismatches:
   - `CIR/MIRSD/24/2011`: PDF date is "15 Dec 2011", gold canonical uses "December 15, 2011" (month format differs after normalize)
   - `Banking Regulation Act`: PDF says "Regulations Act" (plural), gold canonical is "Regulation Act" (legally correct singular)

2. **Resolution metrics are n/a** — no URL resolution is implemented. SEBI URLs are not deterministically derivable from circular identifiers alone; would need a metadata index.

## AI setup

- API key in `.env` as `GEMINI_API_KEY=...` (gitignored)
- Model: `gemini-2.5-flash` (default) — free tier, 10 RPM / 500 RPD
- `gemini-2.0-flash` was deprecated Feb 2026, removed March 2026 — do not use
- Structured output uses `responseJsonSchema` field (renamed from `responseSchema` in older API)
- The extractor auto-loads `.env` via `_load_dotenv()` at module import — no manual export needed

## Design decisions (don't undo without reason)

- **Predictions gitignored, snapshots committed** — predictions are derived; snapshots are the evidence trail
- **AI discovery is a second layer, not a replacement** — regex handles common well-defined patterns; AI catches the long tail (cross-paragraph splits, acts not in the hardcoded list, non-standard phrasing). AI never modifies regex-found records.
- **Generic "Master Circular" alias is kept** — all current PDFs only reference one master circular; removing it breaks page-level recall for alias mentions
- **Bare notifications filtered in adapter, not extractor** — they are real references; we just abstain from scoring them since they have no stable identifier. The raw extractor output still contains them for transparency.
- **No pip install** — all deps vendored under `agent-work/vendor/`; this was intentional for portability
